# Contributing

Contributions are welcome. This is a living resource and will 
improve with community input.

## What you can contribute

- Corrections to technical content
- Additional case studies or examples
- Improvements to clarity or structure
- New regulatory references or jurisdiction coverage
- Code example improvements

## How to contribute

1. Open an issue to describe what you want to change or add
2. Fork the repository
3. Make your changes
4. Submit a pull request with a clear description

## What to keep in mind

- Follow the existing structure and tone
- Cite sources for any new regulatory or research references
- Note if examples are illustrative or based on real data

## Regulatory Content Standards

Regulatory content carries real risk if inaccurate. Any contribution that references legislation, compliance frameworks, or enforcement guidance — such as the EU AI Act, ECOA, or similar — must meet the following standards before it is accepted.

### Source requirements

- Every regulatory claim must include a citation to an official source: the legislation itself, a regulatory body's published guidance, or an official government document
- General references to a law by name are not sufficient. Cite the specific article, section, or provision (for example: EU AI Act, Article 10, paragraph 3)
- Do not paraphrase regulatory requirements without a direct citation

### Accuracy and currency

- Include the date of the source or the version of the legislation you are referencing
- Flag if the referenced regulation is pending, in transition, or subject to ongoing rulemaking
- If a requirement varies by jurisdiction, state that explicitly rather than presenting it as universal

### Review expectations

- Pull requests that add or modify regulatory content will require an additional review pass before merging
- If you are unsure whether a claim is accurate, open an issue first to discuss it before submitting a pull request
